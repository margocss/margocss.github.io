Containers

Containers dictate the width of the content and give the inner-content a slight amount of padding from the outer border. A div with a class of container-full will extend the full width of the page and a div with a class of container-narrow will have a max-width of 960px and will be centered with margin: auto. You can nest a container-narrow inside of a container-full.

Container-narrow

<div class="container-narrow">
   
   <!--your web content here-->

</div>

Grid

Grids must have an outer-div with a class of grid-container:

<div class="grid-container">

The grid-items (columns) are nested inside grid-container and have the following format:

<div class="sm-[n]-x-12 md-[n]-x-12 lg-[n]-x-12">

The default column sizes for three screen-widths:

    sm < 750px,
    md > 750px
    lg > 1000px. 

<div class="grid-container">

   <div class="sm-[n]-x-12 md-[n]-x-12 lg-[n]-x-12">
      <!--your contents here-->
   </div>

</div>

Thus, the grid-item <div class="sm-12-x-12 md-6-x-12 lg-3-x-12"> will be:

  • 100% width ( 12-by-12 columns ) of the container on a small-sized screen (sm, < 750px)

  • 50% width ( 6-by-12 columns ) of the container on a medium-sized screen (lg, > 750px)

  • 25% width (3 by 12 columns) of the container on a large sized screen (md, > 1,000px)

Text Formats

The text right here is inside of a <p> tag. All font sizes and line heights are calculated based on rems. If you want to change the default font-size, you can set a baseline font-size value on the html and all of the typography will scale accordingly. The standard paragraph text is sized at 1rem. And the default line-height is set to 1.625 on tablet/desktop for ideal legibility and 1.25 on mobile to accomodate the lesser screen height.

Forms

Forms need to be inside a <form> tag with a class of form-group:

<form class="form-group">

Forms often contain a pair of <label> + <input> tags. In Margo, these should be encapsulated inside a <div> with a class of form-field 

<form class="form-group">
   <div class="form-field sm-12-x-12 md-4-x-12">
       <label> Username </label>
       <input type="text"/>
   </div>
</form>

Sidebars

Sidebar example demonstrated here on the left.

The Margo Sidebar is a fixed panel on the the left side of the page and is created with
<aside class="sidebar">. It has a view-height of 100% and a default width of 250px. For narrower screens and on mobile devices, it shifts to the top of the page.

In order for the Sidebar to work properly in your layout, you must have <div class="container-full"> immediately after it. Because the sidebar is a major layout component, it should also be near the root of your <body> element. Here is the required HTML structure: 

<aside class="sidebar">
       ...
 </aside>

 <div class="container-full">
       <!--site content-->
 </div>

Sidebar: Code Example
Recommended Use

Note: The use of <div class="nav-spacer"> is a helper class that gives padding if you are using a fixed-navbar.

Also, if you place a <div class="sidebar-content"> inside the sidebar with nested <a class="sidebar-topic"> and <a class="sidebar-subtopic">, the site will provide indentation for a topic-subtopic list of anchor tags (as shown of the sidebar of this very page). 

<aside class="sidebar">
  <!-- Provides padding-top to push content below fixed navbar -->
  <div class="nav-spacer"></div>

  <div class="sidebar-content">
    
    <a class="sidebar-topic" href='#...'>Furniture</a>
    <a class="sidebar-subtopic" href='#...'>Sofas</a>
    <a class="sidebar-subtopic" href='#...'>Desks</a>
    
  </div>
  
</aside>

<!--
  Element will have 250px of padding to

-->

Responsive Padded Images

If you want to create an element with a fixed height, then you can make the image scale to 'cover' the height buy putting the img inside a <div class="img-stretch"/> tag. 

<!-- .example-image {height: 200px; padding: 15px;} -->

<div class="example-image">
    <div class="img-stretch">
       <img src="..."/>
    </div>
</div>
